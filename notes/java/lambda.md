# Lambda выражение
- появились в java 8
- анонимная функция которую можно
- передать как параметр в функцию
- присвоить в переменную 
- вернуть из метода
- вызвать позже

#### Синтаксис
- если есть скобки {} то нужен return
- если их нет и функция однострочная то не надо
```java
(parameters) -> { body }

// без параметров
Runnable task = () -> System.out.println("Hello");
task.run();

// блок кода (нужен return)
Function<Integer, Integer> complex = x -> {
    int result = x * 2;
    result += 10;
    return result;
};
```

- лямбда функция принимает столько параметров сколько принимает **абстрактный метод** соответсвующего **функционального интерфейса**
- то есть стандартно может быть 0, 1 или 2 параметра


# Функциональные интерфейсы
- интерфейс у которого ровно один абстрактный метод
- лямбда выражение это реализация этого абстрактного метода
```java
@FunctionalInterface
public interface MyInterface {
    // ✅ Ровно ОДИН абстрактный метод
    void doSomething(String param);

    // ✅ Можно иметь default методы (не считаются абстрактными)
    default void doSomethingElse() {
        System.out.println("Default");
    }

    // ✅ Можно иметь static методы
    static void staticMethod() {
        System.out.println("Static");
    }

    // ✅ Можно переопределять методы из Object (не считаются)
    @Override
    boolean equals(Object obj);
}
```

## Стандартные функциональные интерфейсы

### 1. Predicate<T>
- проверка условия
- метод  boolean test(T t) возвращает boolen (проверяет условие)

### 2. Function<T, R> 
- преобразование
- R apply(T t) - преобразовывает T в R

### 3. Consumer<T>
- потребитель
- void accept(T t) - принимает Т ничего не возвращает


### 4. Supplier<T>
- поставщик
- T get() - ничего не принимает возвращает Т

- эти интерфейсы для 1 параметра
- для 2 параметров есть интерфейсы с префиксом Bi



##### так же функциональными являются интерфейсы
- **Runnable**
- **Callable**
- **Comparable**
- **Comparator**

#### Лямбда может быть присвоена функциональному интерфейсу
```java
Comparator<String> c = (s1, s2) -> s1.length() - s2.length();
```
- функциональныый интерфейс задает целевой тип лямбды
- @FunctionalInterface аннотация для функционального интерфейса (компилятор проверит точно ли в нем только один абстрактный метод)
- если абстрактный метод функционального интерфейса подходит по параметрам и возвращаемому значению лямбды то он и используется

#### класс Object не функциональный интерфейс
```java
Object o = ()->{}; //не скомпилируется
Runnable  r = () -> {} //скомпилируется
Object o = (Runnable) () -> {}; //скомпилируется
```
#### Раньше вместо лямбд использовались анонимные классы


```java
//приемр сортировки строк при помощи анонимного класса (до появления лямбд)
names.sort(new Comparator<String>() {
    @Override
    public int compare(String s1, String s2) {
        return s1.length() - s2.length();
    }
});
```
- проблема: очень многословно

```java
//с использованием лямбды
names.sort((s1, s2) -> s1.length() - s2.length());
```
## effectively final переменные
- Локальные переменные, которые лямбда захватывает из внешней области видимости, должны быть  effectively final.
- **effectively final** - переменные которые либо уже final либо либо если на них поставить final код все равно будет компилироваться
- то есть это означает что переменная не помечена как final но все равно ей присваевается значение только один раз и дальше она не изменяется
- когда лямбда использует локальную перемнную то она передается по значению (копируется)



# Method reference
- синтаксичекий сахар для лямбд которые просто вызывают один метод
- любую method reference можно переписать как лямбду и наоборот
- обе эти конструкции компилируются в экземпляр функционального интерфейса

```java
//вместо этого
x -> System.out.println(x)
//можно писать это
System.out::println()
```
## виды method reference
1. Ссылка на статический метод
```java
(a, b) -> Math.max(a, b) // lambda
Math::max //method ref
```

2. Ссылка на метод конкретного объекта
```java
x -> System.out.println(x) //lambda
System.out::println //method ref
```

3. Ссылка на метод произвольного объекта определенного типа
```java
s -> s.toUpperCase() //первый аргумент становится объектом, у которого вызывается метод
String::toUpperCase
```
4. Ссылка на конструктор
```java
() -> new ArrayList<>()
ArrayList::new
```
