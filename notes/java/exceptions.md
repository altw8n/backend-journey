# Exceptions (Исключения)
- событие которое происходит во время выполнения и нарушает нормальный поток инструкций.
- исключения это объекты наследуемые от класса Throwable
- класс Throwable наследуется от класса Object

# Object
## Throwable

### Error
- серьезные проблемы JVM (нехватка памяти, переполнение стека)
- не отрабатываем их

### Exception
- **checked**
- ожидаемые проблемы (файл не найден, деление на ноль)
- компилятор требует их обработать


#### RuntimeException
- наследуется от Exception
- **unchecked**
- ошибки программиста
- компилятор не требует их обрабатывать
- баги которые нужно исправлять в коде

#### Синтаксис
- кидаем исключение с помощью throw
```java
throw new ExceptionType("Сообщение об ошибке");
```

- ловим исключения с помощью try, catch, finally
```java
try {
    // Код, который может бросить исключение
    FileReader reader = new FileReader("file.txt");
    int data = reader.read();
} catch (FileNotFoundException e) {
    // Обработка конкретного исключения
    System.out.println("Файл не найден: " + e.getMessage());
} catch (IOException e) {
    // Обработка более общего исключения
    System.out.println("Ошибка ввода-вывода: " + e.getMessage());
} finally {
    // Выполнится ВСЕГДА (даже если было return или исключение)
    System.out.println("Очистка ресурсов");
}
```
- e.getMessage() 
- finally выполняется **всегда** не зависимо от того было исключение или нет


- Порядок catch важен: от более конкретных к более общим
- finally выполняется всегда, кроме случаев:
- Вызов System.exit()
- Падение JVM
- Смерть потока
