# COLLECTIONS HIERARCHY

# Iterateble (interface)
- корнейвой интерфейс в этой иерархии. 
- благодаря ему любой объект (коллекцию) можно обойти с помощью цикла for-each.

## Collection (interface)
- задает контракт API для любой группы объектов (коллекции).
- **add(E e)** 
- **remove(Object O)**
- **clear()**
- **size()**
- **isEmpty()**
- **contains(Object O)**
- **toArray()**
- **equals()**
- **hashCode()**

### List (interface)
- элементы хранятся в порядке в котором были добавлены
- может содержать дубликаты
- может содержать null

#### ArrayList (class)
- внутри обычный массив Object[]
- когда массив полностью заполнен происходит ресайз. реалоцируется новый массив в 1.5 раза больше старого, элементы копируются в него. размер массива растет логарифмически.
##### API
- **get(int index)** - O(1) доступ по индексу
- **add(E el)** - O(1) - вставка в конец (амортизированно) O(n) в худшем случае (реаллокация)
- **add(int index, E el)** - O(n) вставка в середину
- **remove(int index)** - O(n) удаление по индексу
- **remove(Object O)** - O(n) удаление саммого объекта
- **contains(Object O)** - O(n) 
- **size()** , **isEmpty()** - O(1) размер хранится в отдельном поле класса


#### LinkedList (class)
- **двусвязный список**
- **get(int index)** - O(n) доступ по индексу
- **add(E element)** - O(1) вставка в конец
- **add(int index, E element)** - O(n) вставка в середину о(n) из за того что этот индекс сначала нужно найти
- **лучше не использовать для стека и очереди**

### Queue (interface)
- в java включает в себя 1.обычные очереди, 2.дек , 3.очереди с приоритетом

#### PriorityQueue (class)
- очередь с приоритетом
- элементы извлекаются в порядке их приоритета
- внутри реализован как **heap** (куча), по умолчанию используется mtn-heap (наименьший элемент всегда в приоритете)
- **offer(e)** - O(log n)
- **poll()** - O(log n)
- **peek()** - O(1)

### Deque (interface)
- вставка и удаление могут происходить как с головы так и с хвоста
- **offerFirst(e)**
- **offerLast(e)** и т.д.


#### ArrayDeque (class)
- нужно использовать для стека и очереди
- циклический массив с указателями head и tail
- не допускает null
- размер растет логарифмически

##### Stack
- все методы работают за O(1)
- **push(E e)** - добавить на вершину
- **pop()** - извлечь и кдалить с вершины
- **peek()** - посмотреть вершину стека не удаляя
```java
Deque<String> stack = new ArrayDeque<>();
```
##### Queue
- асимптотика О(1) у всех
- **offer(E e)** - добавить в конец
- **poll()** - удалить и извлечь с головы
- **peek()** - посмотреть голову не удаляя
```java
Deque<String> queue = new ArrayDeque<>();
```

### Set (interface)
- множество
- все элементы уникальны


#### HashSet (class)
- реализация хэш-таблица
- обертка над HashMap где вместо ключей заглушки
- порядок элементов при обходе не гарантирован
- **Асимптотика** O(1) для всех методов
- **add(e)**
- **remove(o)**
- **contains(o)** - быстрая проверка через хэш
- **size()**


#### LinkedHashSet (class)
- сохраняет порядок вставки
- внутренее устройство: HashMap + двусвязный список
асимптотика тоже O(1)

### SortedSet (interface)
- остортированное множество

#### TreeSet (class)
- внутри использует TreeMap (RB дерево)
- эелементы всегда отсортированы
- асимптотика O(log n) на все

## Map (interface)
- **находится отдельно в иерархии Iterateble -> Collections..... не участвует** потому что Collections хранить отдельные элементы а Map хранит пары
- Словарь, Карта, ассоциативный массив
- хранит пары (key, value)
- ключи уникальны
- значения могут повторяться

### HashMap
- хэш мапа
- реализация: **хэш-таблица* с **перемешиванием сцеплением**
- асимптотика всех операций O(1) (в худшем случае O(log n))
- требует правильной реализации equals() и hashCode() для ключей
- внутри массив корзин (buckets) - каждая корзина это голова связного списка. при коллизии элемент добавляется в начало связного списка (образуют цепочку). если связные список становится слишком большим то он преобразуется в сбалансированное **RB-дерево**
##### API
- **put(K, V)** - добавить пару
- **get(Object)** - получить значение по ключу
- **remove(Object)** - удалить пару по ключу
- **containsKey(Object)** - проверить наличие ключа
- **containsValue(Object)** - проверить наличие значения
- **keySet()** - получить множество ключей
- **values()** - получить коллекцию значений
```java
Map<String, Integer> map = new HashMap<>();
```

### LinkedHashMap
- HashMap + двусвязный список
- связывает элементы в порядке добавления
- подходит для LRU-cache

### TreeMap
- реализация RB-дерево
- ключи отсортированны
- ключи обязаны реализовывать Comparable для натурального порядка (или нужен Comparator для сравнения)
- асимптотика O(log n)

1. Если нужен список использовать реализацию ArrayList (массив) не использовать LinkedList (двусвязный список, работает медленнее, часто вызывает GC)
2. Использовать интерфейсы для объявления переменных и для параметров метода а не реализации.
```java
Map<String, Integer> map = new HashMap<>();
```
