# Spring Framework
- облегчает внедрение зависимостей (DI)
- облешчает управление жизненным циклом объектов (IoC)


## Какие проблемы он решает?
1. Жесткая связность (зависимости создаются внутри классов через new)

## Inversion of Control (IoC)
- это архитектурный принцип (паттерн программирования)
- управление объектами и их зависимостями передается приложению (фреймфорку) а не осуществляется в коде вручную (не нужно создавать объекты вручную через new)
- это реализуется через ApplicationContext

## Dependency Injection (DI)
- реализация принципа IoC в спринг
- Если у нас есть Класс А, которому для работы нужен Класс Б (это называется зависимость), то вместо того, чтобы Класс А сам создавал Класс Б внутри себя (через new), мы позволяем внешнему источнику (Spring) "внедрить" (inject) готовый Класс Б в Класс А.


## Application Context
- контейнер который управляет жизненным циклом объектов программы (бинами beans) внедрением зависимостей, обработкой аннотаций и конфигураций
- когда запускается spring приложения то сразу запускается ApplicationContext. он сканирует код, находит все классы помеченные аннотациями, создает из них бины связывает и кладет в себя.
- Bean (Бин) — это любой объект, который живет внутри Spring и управляется им.

# задание контекста
```java
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);
```

```java
// 1. Создаем класс конфигурации
@Configuration // Говорит Spring: "Здесь живут настройки"
public class AppConfig {
    
    @Bean // Говорит Spring: "Помести объект, который вернет этот метод, в контейнер"
    public Engine myEngine() {
        return new V8Engine();
    }

    @Bean
    public Car myCar(Engine engine) { 
        // Spring сам найдет бин Engine и передаст его сюда!
        return new Car(engine);
    }
}

// 2. Создаем и используем ApplicationContext
public class Main {
    public static void main(String[] args) {
        // Создаем контейнер и скармливаем ему класс с настройками
        ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);

        // 3. Достаем готовый, полностью настроенный объект из контейнера
        Car car = context.getBean(Car.class);
        
        car.start(); // Работает!
    }
}
```

## способы создания бинов
1. при помощи аннотаций 
- @Component (базовая аннотация для класса)
- @Service
- @Repository и т.д.
2. конфигурационный класс
- класс помеченый аннотацией @Configuration
- в этом классе описываются все бины которые нужно создать
